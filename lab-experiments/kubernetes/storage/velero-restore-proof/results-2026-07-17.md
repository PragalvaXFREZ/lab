## Installation correction

The chart's CRD upgrade hook initially selected the removed image `docker.io/bitnamilegacy/kubectl:1.34` and stopped before installing the controllers. Argo CD already rendered and tracked all 13 chart CRDs, so the duplicate hook was disabled. Argo then applied the CRDs directly with Server-Side Apply and completed the installation.

The first backup attempt ran while `talos-lqv-w4u` briefly became NotReady. The Velero controller on that node lost its API watches and the R2 request ended with a response-body timeout before a `PodVolumeBackup` was created. The failed attempt left 10 objects totaling 4,489 bytes. A Velero `DeleteBackupRequest` removed the failed backup and returned R2 to 0 objects and 0 bytes.

The controller was moved to `talos-opt-7040`, while node agents remained on both workers. The disposable proof pod reattached there through Longhorn and returned the original checksum before the successful retry.
