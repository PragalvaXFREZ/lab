# Architecture decision records

Each file here records one decision: the context it was made in, the choice, and the consequences. The point is to capture the reasoning so a future reader, including a future me, does not have to reverse engineer why the cluster is shaped the way it is.

The format is lightweight, after [Michael Nygard's ADRs](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions). Records are numbered, append only, and immutable: when a decision changes, a new record supersedes the old one instead of editing it.
